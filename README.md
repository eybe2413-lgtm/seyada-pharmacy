# 🏥 صيدلية السيادة | Pharmacie Seyada

نظام إدارة صيدلية متكامل (PWA) مبني بـ React + Vite + Firebase. يعمل على المتصفح مباشرة، قابل للتثبيت على أندرويد وآيفون وسطح المكتب، بدون الحاجة لمتاجر تطبيقات.

---

## 1. التقنيات المستخدمة

| الطبقة | التقنية |
|---|---|
| الواجهة | React 18 + Vite + Tailwind CSS |
| المصادقة | Firebase Authentication |
| قاعدة البيانات | Firebase Firestore |
| الاستضافة | Firebase Hosting |
| الاستضافة | Firebase Hosting |
| PWA | vite-plugin-pwa (تثبيت + عمل دون اتصال للواجهة) |
| اللغات | i18next (عربي RTL افتراضي / فرنسي LTR) |
| الباركود | html5-qrcode (كاميرا) + دعم قارئ USB |
| التصدير | xlsx (Excel) + طباعة المتصفح (PDF) |
| الرسوم البيانية | Recharts |

---

## 2. المتطلبات قبل البدء

- Node.js 18 أو أحدث
- حساب Google لإنشاء مشروع Firebase
- Firebase CLI: `npm install -g firebase-tools`

---

## 3. إعداد مشروع Firebase

1. اذهب إلى [console.firebase.google.com](https://console.firebase.google.com) وأنشئ مشروعاً جديداً.
2. **Authentication** → Sign-in method → فعّل **Email/Password**.
3. **Firestore Database** → أنشئ قاعدة بيانات (وضع الإنتاج Production mode).

5. **Project settings → General → Your apps** → أضف تطبيق ويب (</> ) وانسخ بيانات `firebaseConfig`.

---

## 4. إعداد متغيرات البيئة

```bash
cp .env.example .env
```

افتح `.env` واملأ القيم من الخطوة السابقة:

```
VITE_FIREBASE_API_KEY=...
VITE_FIREBASE_AUTH_DOMAIN=...
VITE_FIREBASE_PROJECT_ID=...
VITE_FIREBASE_STORAGE_BUCKET=...
VITE_FIREBASE_MESSAGING_SENDER_ID=...
VITE_FIREBASE_APP_ID=...
```

---

## 5. التثبيت والتشغيل محلياً

```bash
npm install
npm run dev
```

الموقع سيعمل على `http://localhost:5173`. ملاحظة: مسح الباركود بالكاميرا يتطلب HTTPS أو `localhost` (وهو متوفر تلقائياً أثناء التطوير).

---

## 6. إنشاء أول حساب مدير (Eyba)

النظام **لا يحتوي على أي بيانات دخول مبرمجة مسبقاً** (لا اسم مستخدم ولا كلمة مرور ثابتة) — هذا إجراء أمني مقصود. لإنشاء أول حساب مدير:

1. في Firebase Console → **Authentication → Users → Add user** → أدخل بريد Eyba وكلمة مرور.
2. انسخ الـ **User UID** الذي يظهر.
3. في **Firestore Database → Start collection** → أنشئ مجموعة باسم `users`، وأنشئ مستند بنفس الـ UID المنسوخ، بالحقول التالية:

```
name: "Eyba"
email: "البريد المستخدم"
role: "manager"
active: true
createdAt: (اختر Timestamp → الآن)
```

4. سجّل الدخول بهذا الحساب من التطبيق. من شاشة **المستخدمون** يمكنك بعدها إنشاء حسابي Beden وDahnah مباشرة من داخل التطبيق (صلاحية موظف)، دون الحاجة للرجوع إلى Firebase Console مرة أخرى.

---

## 7. نشر قواعد الأمان والفهارس

```bash
firebase login
firebase use --add   # اختر مشروعك وعدّل .firebaserc إذا لزم
firebase deploy --only firestore:rules,firestore:indexes
```

> **مهم:** بدون نشر `firestore.rules`، ستُرفض كل الطلبات افتراضياً (Firestore تبدأ بقواعد مغلقة).

---

## 8. البناء والنشر على Firebase Hosting

```bash
npm run build
firebase deploy --only hosting
```

سيعطيك رابطاً مباشراً مثل `https://your-project.web.app` — هذا هو رابط التطبيق الجاهز للاستخدام فوراً على أي متصفح، بدون متاجر تطبيقات.

### ربط نطاق مخصص (pharmacie-seyada.com أو seyada.app)

1. Firebase Console → **Hosting → Add custom domain**.
2. أدخل النطاق (مثلاً `pharmacie-seyada.com`).
3. أضف سجلات DNS (TXT للتحقق، ثم A/AAAA أو CNAME) عند مزود النطاق الخاص بك.
4. ستصدر Firebase شهادة SSL مجانية تلقائياً خلال ساعات.

---

## 9. تثبيت التطبيق (PWA)

- **أندرويد (Chrome):** افتح الرابط → قائمة المتصفح (⋮) → "تثبيت التطبيق" / "إضافة إلى الشاشة الرئيسية".
- **آيفون (Safari):** افتح الرابط → زر المشاركة (Share) → "Add to Home Screen".
- **سطح المكتب (Chrome/Edge):** أيقونة التثبيت ⊕ تظهر في شريط العنوان.

بعد التثبيت يعمل التطبيق بأيقونته الخاصة وبدون شريط عنوان المتصفح، تماماً كتطبيق نظامي.

## 10. هيكل المشروع

```
src/
  lib/firebase.js          تهيئة Firebase
  contexts/AuthContext.jsx الجلسة والصلاحيات
  i18n/                    الترجمة (عربي/فرنسي)
  components/              مكوّنات مشتركة (أزرار، نوافذ، القائمة الجانبية...)
  services/                كل منطق Firestore (مبيعات، مشتريات، ديون، رواتب...)
  pages/                   كل شاشة من شاشات النظام
firestore.rules            قواعد الأمان (صلاحيات مدير/موظف)
firestore.indexes.json     الفهارس المركّبة المطلوبة
```

## 11. ملخص الصلاحيات

| الميزة | مدير | موظف |
|---|---|---|
| المبيعات / المشتريات / المخزون / الديون | ✅ | ✅ |
| عرض الأرباح | ✅ | ❌ (محجوبة على مستوى قاعدة البيانات، وليس فقط الواجهة) |
| المحافظ والأرصدة | ✅ | ❌ |
| الرواتب | ✅ | ❌ |
| التقارير | ✅ | ❌ |
| إدارة المستخدمين | ✅ | ❌ |
| سجل العمليات | الكل | عملياته فقط |

هذا الفصل مطبّق فعلياً داخل `firestore.rules` (وليس إخفاءً في الواجهة فقط) — حتى لو حاول موظف الوصول المباشر لقاعدة البيانات فلن يتمكن من قراءة بيانات الأرباح أو الرواتب أو أرصدة المحافظ.

## 12. ملاحظات وحدود معروفة

- **الأرشفة التلقائية** للمبيعات/المشتريات/النفقات الأقدم من سنة تعمل من جهة المتصفح (بدون Cloud Functions)، لذا تتقدم فقط عندما يكون أحد مفتوحاً للتطبيق — وهذا اختيار مقصود لتفادي خطط Firebase المدفوعة (Blaze).
- **مسح الباركود بالكاميرا** يحتاج HTTPS وإذن الكاميرا من المتصفح؛ هو الجزء الأقل اختباراً لأنه يحتاج جهازاً حقيقياً (لم يكن بالإمكان اختباره داخل بيئة البناء هذه).
- **قارئ الباركود USB** يعمل تلقائياً في شاشة المبيعات لأن هذه الأجهزة تُحاكي لوحة المفاتيح: ركّز على حقل البحث وامسح المنتج مباشرة.
- لم يتم تشغيل `npm install` / `npm run build` فعلياً في بيئة الإنشاء هذه (لا تتوفر صلاحية اتصال بالإنترنت هنا)، لكن تم فحص بنية الكود بالكامل (تحليل صياغة JS/JSX، مطابقة كل الاستيرادات لملفاتها، مطابقة كل مفاتيح الترجمة في اللغتين). يُنصح بتشغيل `npm install && npm run dev` فور التحميل والإبلاغ عن أي خطأ تشغيلي إن ظهر.
- تقرير الرواتب السنوي في شاشة التقارير يجمع المبالغ المدفوعة حسب "الشهر الذي يخص الراتب" (مثل راتب يناير) وليس بالضرورة تاريخ الدفع الفعلي إن تأخر السداد لشهر لاحق.
